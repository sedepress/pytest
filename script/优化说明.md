# 电话号码分类脚本 - 优化版使用说明

## 性能优化特点

### 1. 多进程处理
- 使用 `ProcessPoolExecutor` 实现多进程并行处理
- 每个工作进程运行独立的浏览器实例
- 自动检测CPU核心数，默认最多使用4个进程

### 2. 多浏览器窗口
- 每个工作进程拥有独立的Chromium浏览器实例
- 浏览器配置优化：不加载图片、无痕模式
- 自动处理浏览器崩溃和重连

### 3. 并行文件I/O
- 使用 `ThreadPoolExecutor` 并行保存文件
- 减少文件操作等待时间
- 批量处理提高效率

### 4. 智能批处理
- 自动将文件分批处理
- 负载均衡，避免单个进程过载
- 错误恢复机制

## 使用方法

### 基本使用
```bash
python3 script/phone_classifier_optimized.py
```

### 自定义工作器数量
```python
# 在脚本中修改工作器数量
classifier = PhoneClassifierOptimized(max_workers=8)  # 使用8个工作器
```

## 性能对比

### 原版脚本问题
- 串行处理，每次只能检查一个号码
- 单浏览器实例，资源利用率低
- 每次检查都要重新加载页面
- 文件操作串行执行

### 优化版改进
- **并行处理**：多个号码同时检查
- **多浏览器**：充分利用多核CPU
- **连接复用**：减少页面加载时间
- **并行I/O**：文件操作不阻塞主流程

## 预期性能提升

假设处理100个美国电话号码：

- **原版**：约100-150秒（每个号码1-1.5秒）
- **优化版**：约25-40秒（4个工作器并行）

**性能提升：3-4倍**

## 注意事项

1. **内存使用**：每个工作进程会占用约200-300MB内存
2. **浏览器窗口**：会打开多个浏览器窗口，便于调试
3. **网络带宽**：并行请求会增加网络负载
4. **系统资源**：建议在性能较好的机器上运行
